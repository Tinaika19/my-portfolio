Predicting the quality of wine

In this project, I conducted an in-depth analysis and prediction of wine quality using a dataset of various chemical properties of wine samples. Hereâ€™s an overview of the steps and methods applied to achieve robust model performance:

Application Development

Data Pre processing

Missing Value Treatment: Addressed missing values in the "density" and "chloride" features by imputing the mean values for these attributes.

Categorical Conversion:Built a function to convert categorical columns to numerical values, facilitating machine learning model compatibility.

Model Training and Validation

Train-Test Split: Split the dataset into 80% training and 20% testing sets.

Cross-Validation:Applied 10-fold cross-validation on a Decision Tree Classifier, yielding a reliable estimate of model performance across different subsets.

Confusion Matrix Analysis:Evaluated model performance by examining true positives, false negatives, true negatives, and false positives across the 10 folds. sessions.

Performance Metrics

Accuracy: Achieved an average accuracy of 83.52% across folds, with the refined model (after removing "density") achieving 83.98% accuracy, indicating consistent performance. favorite cocktails.

ROC AUCAttained an Area Under the Curve (AUC) of 0.7595 with the original model, which improved to 0.7786 after removing the "density" feature, demonstrating moderate discrimination between wine quality classes.

Outcome and Insights

Removing the "density" feature led to a slight improvement in model accuracy and a stable AUC score, enhancing the model's ability to accurately classify wine quality. This project highlights the importance of feature selection and cross-validation in building effective machine learning models.
